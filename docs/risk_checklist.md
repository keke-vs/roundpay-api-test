# 测试风险清单

这份清单记录我在设计用例时重点关注的风险点。后续如果继续扩展项目，可以按这个清单补自动化用例。

| 模块 | 风险点 | 当前处理方式 | 是否已自动化 |
| --- | --- | --- | --- |
| 登录 | 密码错误仍返回 Token | 校验状态码和响应字段 | 是 |
| 登录 | 用户名或密码为空导致服务异常 | 参数化测试空值 | 是 |
| 转账 | 金额为 0 或负数仍能转账 | 金额边界用例 | 是 |
| 转账 | 金额小数位超过限制 | 传入 1.234 验证拦截 | 是 |
| 转账 | 重复提交导致重复扣款 | 使用同一个 requestId 连续请求 | 是 |
| 转账 | 未登录用户访问转账接口 | 不带 Authorization 请求 | 是 |
| 转账 | 余额不足未拦截 | 使用超大金额验证余额不足提示 | 是 |
| 个人信息 | 手机号格式错误仍保存 | 参数化手机号异常值 | 是 |
| 个人信息 | 未登录用户修改资料 | 不带 Authorization 请求 | 是 |
| 基金 | 关闭状态基金仍可购买 | 购买 FUND002 验证失败 | 是 |
| 基金 | 低于起购金额仍可购买 | 购买金额 1 元验证失败 | 是 |
| 基金 | 基金不存在时返回不明确 | 传入 FUND999 验证错误提示 | 是 |
| 基金 | 接口字段变化影响前端 | jsonschema 校验核心字段 | 部分 |
| 性能 | 基金列表一次性返回过多数据 | JMeter 记录压测思路 | 部分 |
