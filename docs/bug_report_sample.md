# 缺陷记录示例

## BUG-001 基金购买页面响应时间偏高

- 严重程度：P2
- 所属模块：基金购买
- 测试环境：本地 mock 环境
- 发现方式：JMeter 聚合报告

### 前置条件

用户已登录，基金列表存在多条数据。

### 复现步骤

1. 使用 JMeter 对基金购买链路发起并发请求
2. 观察基金购买页面关联接口的平均响应时间
3. 对比登录、转账等接口响应耗时

### 实际结果

基金购买相关接口响应时间明显高于其他接口，页面等待时间较长。

### 预期结果

基金购买页面应在可接受时间内返回，不影响用户继续操作。

### 初步分析

接口一次性返回基金信息较多，前端展示时需要处理的数据量偏大。

### 建议方案

将基金列表改为分页加载，减少单次接口返回数据量；优化后重新执行回归测试，确认购买流程和页面展示没有引入新问题。
