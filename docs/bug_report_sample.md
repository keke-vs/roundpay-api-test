# 缺陷记录样例

## BUG-001 基金购买接口响应时间偏高

- 严重程度：P2
- 所属模块：基金购买
- 测试环境：本地 mock 环境
- 发现方式：JMeter 聚合报告

### 前置条件

用户已登录，基金列表存在多条数据。

### 复现步骤

1. 使用 JMeter 对基金购买链路发起并发请求
2. 观察基金购买页面关联接口的平均响应时间
3. 对比登录、转账等接口响应耗时

### 实际结果

基金购买接口在并发请求下响应时间波动较明显，最大响应时间高于平均响应时间较多。

### 预期结果

基金购买接口应保持稳定响应，不影响用户提交购买操作。

### 初步分析

购买链路可能同时涉及基金状态、起购金额、账户余额和交易记录。当前是 mock 环境，暂时只能从接口处理逻辑上判断；如果接入真实数据库，需要继续检查查询条件、索引和事务处理。

### 建议方案

基金列表采用分页加载，减少单次返回数据量；购买接口增加请求幂等控制；接入真实数据库后，重点检查账户表、基金表和交易流水表的索引。优化后重新执行接口回归和性能测试。
