## 本次观察

- 登录接口响应结构稳定，成功响应中包含 Token、账号和昵称
- 前端系统可直接通过 mock 服务访问，页面资源由同一个 Python 服务托管
- 未登录时只展示登录页，登录成功后才进入业务工作台
- 账户概览接口能返回账号、昵称、手机号和余额，用于支撑前端展示
- 转账和基金购买会生成交易流水，前端首页和流水页可查询
- 转账接口的重复提交用例通过，同一个 `requestId` 没有产生第二次扣款结果
- 金额为 0、负数和超过两位小数时均被拦截
- 基金购买接口能区分开放基金和关闭基金
- 余额不足、基金不存在和未登录修改资料场景均能返回明确错误
- 当前测试仍缺少数据库校验，余额变化主要通过接口返回值判断
