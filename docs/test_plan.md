# 圆小付系统测试计划

## 1. 测试目标

验证圆小付系统核心接口是否满足基本业务要求，重点关注登录鉴权、转账资金变化、个人信息修改、基金购买和接口返回结构稳定性。

## 2. 测试范围

本次测试覆盖以下模块：

- 登录接口
- 转账接口
- 个人信息修改接口
- 基金列表查询接口
- 基金购买接口

暂不覆盖内容：

- 第三方支付通道回调
- 短信验证码真实发送
- 银行卡实名认证
- 账务日终清算

## 3. 测试方法

- 功能测试：根据需求拆分正常流程和异常流程
- 接口测试：使用 Postman 验证接口请求、响应和状态码
- 自动化测试：使用 pytest + requests 编写回归用例
- 结构校验：使用 jsonschema 校验核心接口响应字段
- 性能测试：使用 JMeter 对基金购买链路做基础压测

## 4. 重点风险

- 转账成功后付款方扣款但收款方未到账
- 重复提交导致多次扣款
- 未登录用户可以访问敏感接口
- 金额边界处理不正确
- 基金购买接口一次性返回数据过多，页面响应变慢

## 5. 准出标准

- 核心流程用例全部通过
- P0/P1 级缺陷全部修复并回归通过
- 自动化回归用例通过率达到 100%
- 关键接口响应结构符合前端解析要求
